1.  Convert a BPC skeleton Schematron script into one tailored for a process. - bpcprocess2sch.xsl

Filename: bpcprocess2sch.xsl

$Id$

The tailoring is simple in the attributes, text, and comments.

1.1. Invocation parameters and input file

The input file is the image of the output file with placeholders of the format "{bpc:process}" that get replaced in-situ in order to create the output file.

Comments and text get both the process identifier and the version.

Attributes and filenames are replaced with only the process identifier.

Schema skeleton to be fleshed out.

Schema skeleton to be fleshed out.

Main input semantics file.

antDynamicScriptURI required="yes" as="xsd:string" (xsl:param)

The location in which to create the dynamic Ant script.

BPCversion required="yes" as="xsd:string" (xsl:param)

The version number not including a preceding "v".

dateTime required="yes" as="xsd:string" (xsl:param)

The date and time stamp to distinguish releases of a particular version.

For the convenience of human readers, the outputs are indented

stylesheet-prefix="xslo" result-prefix="xsl" (xsl:namespace-alias)

Portions of the output are in the XSLT vocabulary, thus this is needed.
