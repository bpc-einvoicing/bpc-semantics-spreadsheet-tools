1.  Convert a UBL genericode model into a BPC semantic model XML instance - ublgc2bpcgc.xsl

Filename: ublgc2bpcgc.xsl

$Id$

The BPC Semantics group has imposed changes in cardinality and has specified these values in a spreadsheet. The spreadsheet is translated to a colloquial XML for downstream processing.

This stylesheet reads the genericode to augment a copy of the supplied UBL spreadsheet with columns used by a schema subset tool.

indent="yes" (xsl:output)

Indent the results

elements="ublcardinalities" (xsl:strip-space)

It helps not to have white-space characters for comparisons

1.1. Invocation parameters and input file

The input file is the BPC Semantics Spreadsheet with all of the business rule information that contains cardinality requirements for the BPC subset schemas for each customization.

ublgc required="yes" as="document-node()" (xsl:param)

This is the UBL document model genericode file passed as a document node which when using Saxon uses the command-line argument +ublgc={uri}

embedded-comment required="yes" as="xsd:string" (xsl:param)

The comment to embed at the beginning of the output XML

1.3. Genericode access functions

Various convenience functions to access information from an instance of genericode

Tracking the rows by the unique dictionary entry name

bpc:col(2) as="element(SimpleValue)*" (xsl:function)

Returning a row's column value by its column name

Parameter row as="element(Row)"

The genericode row

Parameter column as="xsd:string"

The genericode column

bpc:checkConstraints(2) as="xsd:boolean" (xsl:function)

Returning the allowed constraint of a constraint

Parameter old as="xsd:string"

The old constraint

Parameter new as="xsd:string"

The new constraint

List of all of the supplemental components
